What is ORC ?

ORC is the abbreviation for (O)pen (R)agnarok (C)lient.

It's what the name suggests, an Open Source, cross-platform Ragnarok Client emulator written in C/C++. OpenGL is used to render the graphics and SDL is used for everything else.

The main goal is to make it smooth playable. But hopefully it will get as complete as the original client. It is planned that the network handling will be done by Openkore and that the client is controlable by it too.

Can i help with the development ?

Yes. New developers are always welcome to the Openkore project, and as this is related to Openkore the same guidelines apply. The code is written in C/C++ using Code::Blocks as IDE, and it is recommended that you use it too.

Installing Code::Blocks on Windows

Download Code::Blocks IDE, with MINGW compiler and install it to "C:\CodeBlocks" for example.
back to top

Required dependencies

In Code::Blocks, open the DevPak plugin, set the directory for installations to "C:\CodeBlocks". Download and install the following devpaks, in that order:
  • zlib
  • libjpeg
  • libpng
  • libtiff
  • SDL
  • SDL_Sound
  • SDL_Mixer
  • SDL_Image
  • libgrf
back to top

Compiling the Code

In Code::Blocks, open the SVN plugin and checkout from "https://svn.sourceforge.net/svnroot/openkore/orc/trunk" into some folder on your drive, e.g. "C:\OrcSVN"

Go into that folder and open the "Orc.cbp", it should compile without any problems.

What needs to be done ?

  • Objects show up in the wrong positions
  • Use "grf_chunk_get" in the CGRF_Interface
  • Identify the unknown members in file structures
  • Implement a GUI
  • RO text file parser (e.g. resnametable.txt)
  • Show a loading screen
  • Use libsprite (from hongli) to load and display sprites
  • Integrate Openkore (IPC or API ?)
  • Implement texture manager
  • Implement memory manager
